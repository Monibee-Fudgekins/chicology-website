## Quick Start

1. **Deploy the Worker**: `npx wrangler deploy`
2. **Set secrets**: GitHub token, AI gateway URL, run key
3. **Automation**: Cloudflare cron triggers run daily (no GitHub Actions schedules or manual triggers)
4. **Connect Cloudflare Pages**: For static hosting

> Everything else manages itself automatically.

## Architecture

- **Frontend**: Next.js 16 + TypeScript + Tailwind (static export)
- **Backend**: Cloudflare Workers + AI Gateway  
- **Content**: MDX blog posts generated by AI
- **Hosting**: Cloudflare Pages (static export)
- **Automation**: Cloudflare cron triggers + Worker only (GitHub Actions optional for CI checks)

## Key Endpoints

- `/__health` - System status and metrics
- `/metrics` - Prometheus metrics (optionally bearer-protected)
- `/__status` - Recent run summary
- `/__design-status` - Design iteration status

## Health Monitoring

The system tracks AI operation success rates with automatic alerting:
- **GOOD**: Success rate >= 85%
- **WARNING**: Success rate < 85% but >= 60%
- **CRITICAL**: Success rate < 60%

See [HEALTH.md](./HEALTH.md) for detailed monitoring documentation and troubleshooting.
